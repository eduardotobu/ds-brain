Something that these tasks defined in the [[Machine Learning and Prediction]] note have in common, is that they are hard to solve in a traditional engineering way where we specify rules or solutions directly to the problem. **This is much easier to learn from experience**.

Machine learning tasks are really provided in terms of examples. 
It's very easy to show examples of how you would want the method to behave, examples of correct behavior. 

So in supervised learning we are given an example along with a target, and the goal of the machine learning algorithm is to find out how to produce the target from the example.

More concrete in machine learning, **we hypothesize a collection of functions (or mappings) parametrized by a parameter, from the example to the targets. The machine learning algorithm then automates the process of finding the parameter of the function that fits with the example-target pairs the best.**

So after training the model with the examples-targets, we obtain a model with new parameters that fits with the example-target pairs the best, so it can be applicable with new examples.

So, lets recall: 
In traditional programming, we use data and a set of rules to **produce an outcome**. 
In Machine Learning we use data and a set of outcomes to **produce the rules**.
